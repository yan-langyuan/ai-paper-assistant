## 二、产品目标

### 2.1 用户目标

| 目标 | 衡量指标 | 目标值 |
|------|----------|:---:|
| 降低文献阅读时间 | 单篇文献从获取到理解的时间 | 从平均45分钟降至15分钟 |
| 消除参考文献格式错误 | 格式化后的文献条目一次通过率 | >95% |
| 降低降重挫败感 | 用户改写后重复度下降幅度 | 平均下降40-60% |
| 保障用户隐私 | 本地处理覆盖率（可离线使用的功能比例） | 降重检测100%本地 |

### 2.2 业务目标

| 目标 | 衡量指标 | 目标值 |
|------|----------|:---:|
| 用户验证 | MVP上线首月注册用户 | ≥500人 |
| 核心留存 | 注册后7日留存率 | ≥40% |
| 功能验证 | P0功能使用率达50%以上的功能数 | ≥4/6 |
| NPS | 用户净推荐值 | ≥30 |

### 2.3 非目标（MVP明确不做）

- ❌ 不做全文自动生成
- ❌ 不做一键全文降重
- ❌ 不做正式查重报告
- ❌ 不做英文论文全流程支持
- ❌ 不做移动端（小程序/App）
- ❌ 不做付费变现系统

## 四、功能详述

### 4.1 模块一：智能文献助手

#### F01: 文献导入与解析

**功能描述**：用户通过上传PDF文件、粘贴链接或手动输入，系统解析并提取文献元数据。

**操作逻辑**：
1. 用户进入文献库页面，点击"导入文献"
2. 选择导入方式：本地上传PDF / 粘贴链接 / 手动录入
3. 【本地上传】拖拽或选择PDF文件，支持批量选择（上限10篇）
4. 【粘贴链接】输入知网/万方/DOI链接，系统抓取元数据
5. 上传后进入解析状态（参4.1加载状态设计）
6. 解析完成后，元数据自动填入，文献出现在文献库列表

**页面反馈**：
- 文件格式校验失败：Toast"仅支持PDF格式" + 上传区红色边框
- 文件>50MB：Toast"文件最大50MB，当前XX MB"
- 批量上传：显示进度"正在上传 3/10""正在解析 3/10"
- 解析失败：该文献标注"解析失败"状态，点击可查看原因+手动录入入口
- 解析成功：文献出现在列表顶部 + 自动触发摘要生成

**异常处理**：
| 异常场景 | 处理方式 |
|------|------|
| 扫描版PDF | 提示"不支持扫描版PDF，请上传文字版PDF" |
| 加密PDF | 提示"该PDF已加密，无法解析" |
| 损坏PDF | 提示"文件已损坏，请重新上传" |
| 网络中断(上传中) | 自动续传，失败后"上传中断，点击重试" |
| 网络中断(解析中) | 解析在服务端完成，失败后标记"解析失败"+"点击重试" |
| DOI API超时 | 使用已提取的元数据，DOI字段留空，不阻塞整体流程 |

**验收标准**：
- [x] 支持上传PDF文件（≤50MB），15秒内完成文本提取
- [x] 批量上传10篇，30秒内全部解析完毕
- [x] 输入DOI链接，自动补全元数据（作者/标题/期刊/年份）
- [x] 解析失败文献不阻塞其他文献解析
- [x] 扫描版/加密/损坏PDF，给出明确提示

---

#### F02: 文献结构化摘要生成

**功能描述**：AI自动生成五模块结构化摘要卡片，帮助用户快速理解文献核心内容。

**操作逻辑**：
1. 文献解析完成后，自动触发摘要生成
2. AI按固定模板输出：研究背景(1句) → 研究问题(1句) → 研究方法(1句) → 核心发现(1-3点) → 研究局限(1句)
3. 摘要卡片展示在文献详情页
4. 用户可复制摘要全文（点击"复制摘要"按钮）

**页面反馈**：
- 摘要生成中：摘要卡片区域显示骨架屏 + "正在生成摘要..."
- 摘要生成失败：卡片显示"摘要生成失败" + [重试]按钮
- 摘要生成成功：卡片从骨架屏渐变为实际内容（淡入动画，0.3s）
- 复制成功：Toast"摘要已复制"

**异常处理**：
| 异常场景 | 处理方式 |
|------|------|
| AI服务超时(>30s) | 前端显示"生成时间较长，请稍候..." + 取消按钮 |
| AI返回格式异常 | 降级显示为纯文本摘要（不分区模块） |
| 文献全文过短(<500字) | 摘要生成质量可能不足，标注"原文较短，摘要仅供参考" |

**验收标准**：
- [x] 解析完成后30秒内生成结构化摘要
- [x] 摘要卡片包含五个固定模块
- [x] 摘要底部强制显示"AI生成摘要，仅供参考，请核实原文"
- [x] 支持一键复制摘要全文
- [x] 摘要生成失败可重试

### 4.2 模块二：格式排版助手

#### F03: 参考文献一键格式化

**功能描述**：从文献库选择文献或手动输入，一键生成符合GB/T 7714/APA/MLA标准的参考文献列表。

**操作逻辑**：
1. 用户从文献库勾选需要引用的文献(支持单选/多选)
2. 点击"格式化引用"进入格式预览页
3. 选择目标格式标准（默认GB/T 7714-2015）
4. 系统即时渲染格式化后的参考文献列表
5. 选择排序方式（拼音/字母/引用顺序）
6. 用户可对单篇文献切换格式标准（如英文文献用APA）
7. 点击"导出Word"生成.docx文件并下载

**页面反馈**：
- 无文献选择时："格式化引用"按钮置灰
- 格式预览：切换标准后预览即时刷新（前端渲染，无网络请求）
- 单篇切换标准：该条目右侧显示当前格式标签
- 导出成功：Toast"导出成功" + 浏览器自动下载
- 导出失败：Toast"导出失败，请重试" + [重试]按钮
- 元数据不完整：该条目显示黄色背景 + "信息不完整，点击补全→"

**异常处理**：
| 异常场景 | 处理方式 |
|------|------|
| 文献元数据缺少作者 | 该条目显示"佚名"或"Anon." |
| 文献元数据缺少年份 | 该条目显示"[年份不详]"或"n.d." |
| 文献元数据缺少期刊 | 该条目标注"文献信息不完整"，黄色高亮 |
| 未知文献类型 | 默认按期刊[J]格式处理，标注"类型待确认" |
| 导出生成失败 | 提示"导出失败"，重试1次，仍失败建议"复制文本手动粘贴" |

**验收标准**：
- [x] 支持GB/T 7714-2015、APA 7th、MLA 9th三种标准
- [x] 30条文献格式化+导出Word，30秒内完成
- [x] 切换标准后预览即时刷新（<100ms）
- [x] 单篇文献可切换格式标准
- [x] 文献信息不完整时有明确视觉提示
- [x] 导出Word文档包含正确的参考文献格式

### 4.3 模块三：降重改写助手（核心差异化）

#### F04: 段落重复检测与高亮

**功能描述**：粘贴待检测文本，系统在本地进行语义比对，三色高亮标注疑似重复段落。

**操作逻辑**：
1. 用户粘贴文本（500-3000字）
2. 点击"开始检测"
3. 系统在本地进行语义向量化+相似度比对
4. 返回三色标注结果：红(>80%)、黄(50-80%)、绿(<50%)
5. 每句高亮附带改写方向建议(1句话)

**页面反馈**：
- 输入<500字：按钮置灰 + 提示"至少输入500字"
- 输入>3000字：按钮置灰 + 字数变红 + 提示"超过3000字限制"
- 检测中：按钮显示loading + "检测中...预计5秒"
- 检测完成：从顶部开始逐句淡入高亮效果(动效)
- 红色/黄色句默认展开，绿色句默认折叠
- 检测结果页顶部显示摘要统计（红N处,黄N处,绿N处）

**异常处理**：
| 异常场景 | 处理方式 |
|------|------|
| 输入全为空格/换行 | 过滤空白后字数<500，按钮置灰 |
| 输入包含特殊字符/乱码 | 正常处理，不做特殊过滤 |
| 本地模型加载失败 | 提示"检测引擎加载失败，请刷新页面重试" + [刷新]按钮 |
| 检测耗时>10秒 | 显示"检测时间较长，请稍候..." |

**验收标准**：
- [x] 2000字检测<5秒
- [x] 三色高亮(红/黄/绿)逻辑正确
- [x] 每句附带改写方向建议
- [x] 页面明确提示"非正式查重结果"
- [x] 断网状态检测仍可用（验证本地处理）
- [x] 输入字数校验正确(500-3000)

---

#### F05: 逐句改写建议（3方案）★ 核心差异化

**功能描述**：针对高亮重复句，生成三个梯度的改写方案，用户自主选择和编辑。**不提供一键改写全文**。

**操作逻辑**：
1. 用户在检测结果页点击某个高亮句的"查看改写建议"
2. 右侧滑入改写方案抽屉
3. 展示3个改写方案：方案A(保守)、方案B(均衡)、方案C(激进)
4. 每个方案用Diff颜色标注改了什么(绿=保留术语/黄=调整/红=重写)
5. 用户选择一个方案，点击"应用此方案"
6. 改写内容填入编辑器，用户可自由修改
7. 点击"重新检测"验证改写效果
8. 形成 检测→改写→再检测 闭环

**页面反馈**：
- 抽屉从右侧滑入动画(0.3s ease-out)
- 方案加载中：3个方案卡片显示骨架屏
- 方案生成失败：该方案卡片显示"生成失败，点击重试"
- 方案C：顶部有⚠️图标 + tooltip"该方案改动较大，请核实学术含义是否变化"
- 应用方案后：抽屉关闭，高亮句状态变为"已改写"
- "重新检测"后：高亮句颜色可能降低(红→黄→绿)，显示新旧对比
- AI占比超30%：弹窗提醒(参F06)

**异常处理**：
| 异常场景 | 处理方式 |
|------|------|
| AI改写服务超时 | 单个方案生成失败，其他方案正常展示 |
| AI返回格式异常 | 显示原始AI返回结果 + 提示"格式异常，可手动修改" |
| 句子过短(<10字) | 仅生成1个改写方案(方案B)，标注"句子较短，提供1个方案" |
| 句子过长(>200字) | 自动拆分长句，逐段生成改写方案 |

**验收标准**：
- [x] 每句生成3个改写方案(保守/均衡/激进)
- [x] 方案A/B/C改动幅度递增，Diff颜色标注正确
- [x] 方案C附带学术含义变更提醒
- [x] 改写建议生成<3秒/句
- [x] 用户编辑后可"重新检测"
- [x] **无"一键改写全文"按钮**
- [x] 改写后AI占比计算正确

#### F06: 写作风险预警

**功能描述**：监控AI功能使用比例，在适当阈值弹出学术诚信提醒。

**操作逻辑**：
1. 系统在前端记录每次AI操作的字数
2. 计算AI辅助字数占文档总字数的比例
3. <20%：无提示
4. 20-30%：侧边栏浅黄提示，不阻塞操作
5. 30-50%：弹窗温和提醒
6. >50%：导出前强制弹出学术诚信确认弹窗(需勾选确认)

**页面反馈**：
- 侧边栏提示：从右侧滑入，停留5秒后自动缩小为图标
- 弹窗提醒(30-50%)：居中模态弹窗，点击"知道了"关闭
- 强制确认(>50%)：居中模态弹窗 + 勾选框"我已阅读学术诚信提醒" + [确认]
- 提示文案风格：建议型，不用"警告""禁止"等词语

**异常处理**：
| 异常场景 | 处理方式 |
|------|------|
| 清除浏览器数据 | AI计数器重置，重新计算 |
| 多设备使用 | 各设备独立计数（数据仅存储在本地） |

**验收标准**：
- [x] 20%/30%/50%三个阈值触发正确
- [x] 提示文案无说教感
- [x] 20%阈值提示不阻塞操作
- [x] >50%导出前强制确认
- [x] 计数器数据存储在本地(不清除浏览器数据则持续累计)

## 七、交互说明

### 7.1 全局交互规范

| 操作类型 | 反馈时效 | 反馈形式 |
|------|:---:|------|
| 页面跳转 | <2秒 | 路由过渡动画(0.2s fade) |
| 按钮点击 | <200ms | 按钮按下态 → 恢复正常态 / loading态 |
| AI生成(摘要/改写) | <30秒 | 骨架屏 / 进度文案 / 转圈动画 |
| 本地计算(格式化/检测) | <5秒 | 按钮loading态 |
| 数据保存/导出 | <3秒 | Toast成功/失败提示 |

### 7.2 关键设计约束

| 编号 | 约束内容 | 理由 |
|:----:|------|------|
| DC-01 | 降重模块**禁止**提供"一键改写全文"按钮 | 核心差异化定位，用户调研结论 |
| DC-02 | 所有AI生成内容**必须**标注"AI生成，仅供参考" | 学术合规 |
| DC-03 | 降重检测**必须**本地完成，文本不传服务器 | 用户隐私顾虑(8.5%) |
| DC-04 | 改写方案以"建议"形式呈现，不自动替换原文 | 保留用户掌控感 |
| DC-05 | 风险预警**仅提示不拦截**，文案免说教 | 用户体验，非惩罚性质 |

### 7.3 异常场景覆盖

| 编号 | 场景 | 处理方式 | 负责页面 |
|:----:|------|------|:---:|
| E01 | 扫描版PDF上传 | 提示不支持，建议上传文字版 | P03 |
| E02 | 文件>50MB | 提示超限，显示当前大小 | P03 |
| E03 | 加密/损坏PDF | 提示文件异常，无法解析 | P04 |
| E04 | 网络异常(上传中) | 自动续传，失败提示重试 | P03 |
| E05 | AI服务超时(摘要) | 显示超时提示+重试按钮 | P05 |
| E06 | AI服务超时(改写) | 单方案失败，其他正常 | P10 |
| E07 | 输入为空 | 按钮置灰，无法提交 | P08 |
| E08 | 输入>3000字 | 按钮置灰+红色提示 | P08 |
| E09 | 本地模型加载失败 | 提示刷新页面重试 | P09 |
| E10 | 元数据缺失(格式化) | 黄色标注+引导补全 | P07 |
| E11 | 导出生成失败 | 提示重试+降级方案(复制文本) | P07 |
| E12 | 删除操作 | 二次确认弹窗 | P17 |

## 九、非功能需求

### 9.1 性能要求

| 指标 | 要求 | 测试条件 |
|------|------|------|
| 页面首屏加载 | <2s | Chrome, 4G网络 |
| PDF解析 | <15s/篇 | 20页纯文本PDF |
| 摘要生成 | <30s/篇 | 含AI调用 |
| 重复检测 | <5s/2000字 | 本地处理 |
| 改写方案生成 | <3s/句 | 含AI调用 |
| 格式预览刷新 | <100ms | 纯前端，切换标准 |
| 并发支持 | 50用户同时使用 | 云端AI调用 |

### 9.2 安全要求

| 要求 | 实现方式 |
|------|------|
| 用户密码加密存储 | bcrypt哈希 |
| 文献内容隔离 | 用户间文献数据隔离，不可跨用户访问 |
| 降重文本本地处理 | 语义向量化在浏览器端完成(WebAssembly) |
| HTTPS强制 | 全站HTTPS，HTTP自动跳转 |
| XSS防护 | 输入内容HTML转义，CSP策略 |
| 无第三方跟踪 | 不使用Google Analytics等第三方跟踪脚本 |

### 9.3 可用性要求

| 要求 | 目标值 |
|------|:---:|
| 首次任务完成率 | >80%（新用户首次导入文献并查看摘要） |
| 核心任务效率 | 从登录到获得第一篇摘要<3分钟 |
| 错误恢复率 | >90%（用户遇到错误后能通过重试/返回恢复正常） |
| WCAG无障碍 | AA级（颜色对比度、键盘导航、屏幕阅读器） |

### 9.4 数据需求

| 数据 | 存储策略 | 备份策略 |
|------|------|------|
| 用户账号信息 | PostgreSQL, 加密存储 | 每日全量备份 |
| 文献元数据+全文 | PostgreSQL + 对象存储(COS/OSS) | 每日全量备份 |
| AI摘要内容 | PostgreSQL | 每日全量备份 |
| AI计数器(占比) | 浏览器localStorage | 无备份(隐私设计) |
| 降重/改写文本 | **不存储**（仅存在于用户会话） | 无存储 |
